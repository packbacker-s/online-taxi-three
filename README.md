# online-taxi-three
网约车
                       微服务设计原则
架构、模式、拆分、隔离
目标：隔离系统的变化点
具体原则：
①高内聚、低耦合
高内聚：简单指功能模块集中在一起
低耦合：减少类内部、成员方法调用另一个成员方法
②高度自治：开发、测试、构建、部署、运行、发布
③以业务为中心
④弹性设计：容错、隔离、降级
⑤自动化：持续集成、持续支付
⑥粒度把控：（无具体标准要多细）



AKF
x轴：水平复制，解决单点故障
y轴：业务分层，把A拆分
z轴：数据分片
